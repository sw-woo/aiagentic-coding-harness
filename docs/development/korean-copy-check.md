# 한글 띄어쓰기 점검 장치

작성일: 2026-04-07

이 저장소는 한국어 문장이 많기 때문에, 작성 후 한글 띄어쓰기를 한 번 더 보는 장치가 필요합니다.

## 현재 적용 방식

로컬 점검 스크립트:

```bash
npm run spacing:ko
```

이 스크립트는 다음을 수행합니다.

- `src/app`, `src/components`, `docs` 안의 한국어 문장이 있는 라인을 찾습니다.
- KSS의 `correct_spacing` 기능으로 추천 띄어쓰기를 계산합니다.
- 원문과 추천문이 다를 때만 보고합니다.
- 자동 수정은 하지 않습니다.

## 왜 자동 수정이 아닌가

한글 띄어쓰기는 문맥 의존성이 커서, 기계가 무조건 고치는 방식은 오히려 잘못된 문장을 만들 수 있습니다.
이 사이트는 사실 기반과 문장 품질을 모두 중요하게 보기 때문에, 우선은 `점검 보고서` 방식으로만 둡니다.

## 의존성

스크립트는 Python 패키지 `kss`가 필요합니다.

설치:

```bash
python3 -m pip install kss
```

공식 저장소:
- https://github.com/hyunwoongko/kss

## 추가로 검토할 만한 장치

이번 조사 기준으로는 다음 조합이 현실적입니다.

### 1. KSS

- 용도: 로컬 배치 점검, 띄어쓰기 추천
- 장점:
  - 최근 기준으로도 유지보수 이력 확인 가능
  - `correct_spacing` 기능을 공식적으로 제공
  - 현재 저장소에 이미 적용한 스크립트와 잘 맞음
- 한계:
  - 자동 수정은 과감하게 하면 오탐이 날 수 있음
  - 최종 문장 판단은 사람이 해야 함

### 2. korean-language-server

- 저장소: https://github.com/aca/korean-language-server
- 용도: 에디터 안에서 맞춤법/문법 보조
- 장점:
  - LSP 기반이라 여러 에디터에 붙이기 쉬움
  - 문장 작성 중 즉시 피드백을 받기 좋음
- 한계:
  - 프로젝트 공통 CI 장치라기보다는 개인 편집기 보조에 가까움
  - 팀 전체 표준으로 강제하기는 어려움

### 3. py-hanspell

- 저장소: https://github.com/ssut/py-hanspell
- 용도: 네이버 맞춤법 검사기 기반의 한글 교정
- 장점:
  - 교정 품질 체감이 좋은 편
  - 띄어쓰기 + 맞춤법 교정을 같이 봄
- 한계:
  - 외부 서비스 의존
  - 장기 자동화 파이프라인에 바로 넣기엔 조심해야 함

### 4. hunspell 한국어 사전

- 저장소: https://github.com/spellcheck-ko/hunspell-dict-ko
- 용도: 시스템 / 에디터 / 애플리케이션 단위의 한국어 맞춤법 검사 기반
- 장점:
  - 오픈소스 생태계에서 널리 쓰이는 한국어 사전 기반
  - macOS, Linux, Firefox, LibreOffice 같은 환경과 연결 가능
  - 팀 표준 에디터 환경에 녹이기 좋음
- 한계:
  - 띄어쓰기 추천 엔진이라기보다 맞춤법 사전 기반에 가까움
  - 이 저장소처럼 긴 설명 문장을 다듬는 보조로는 KSS나 py-hanspell 쪽이 더 직접적일 수 있음

## 현재 권장 조합

이 저장소에는 아래 조합이 가장 현실적입니다.

1. 프로젝트 공통:
   - `KSS` 기반 배치 점검 스크립트

2. 개인 편집 환경:
   - `korean-language-server` 같은 에디터 보조

3. 선택적 수동 점검:
   - 필요 시 `py-hanspell`

4. 편집기 / OS 기반 보조:
   - `korean-language-server`
   - `hunspell-dict-ko`

## MCP에 대해서

이번 조사 기준으로는, 한국어 띄어쓰기/문장 교정에 특화되어 널리 쓰이는 성숙한 MCP 서버는 확인하지 못했습니다.
즉 현재는:

- 한국어 문장 보조는 `MCP`
- 한국어 문장 교정은 `로컬 스크립트 / 에디터 보조`

로 분리해서 보는 편이 현실적입니다.
